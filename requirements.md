# Requirements Document: Kanban-style Backlog Management Tool

## 1. 概要
PythonとStreamlitを用いた、カンバン方式のバックログ管理ツール。

## 2. 機能要件
- **タスク追加機能**:
  - タイトル、優先度（高・中・低）、担当者を指定して新しいタスクを作成する。
  - 新規作成されたタスクは初期状態として「To Do」ステータスを持つ。
- **ステータス変更機能**:
  - 各タスクのステータスを「To Do」「Doing」「Done」の間で任意に変更できる。
- **タスク削除機能**:
  - 任意のタスクを削除できる。
- **データ永続化**:
  - タスクデータはローカルの `tasks.db` (SQLite) に保存する。
  - アプリケーション再起動時にもデータを読み込み、以前の状態を完全に維持する。
- **進捗の定量化 (Visual Metrics)**:
  - 各ステータス（To Do, Doing, Done）ごとのタスク数を集計する。
  - サイドバーまたはメイン画面上部に「進捗率（Doneの割合）」と「ステータス分布」を数値または簡単なグラフで表示する。
- **データエクスポート (CSV Export)**:
  - 登録されている全タスクをCSV形式でダウンロードできる機能をサイドバーに配置する。

## 3. UI/UX要件 (ユーザー操作フロー)
1. **メイン画面表示**: 画面は「To Do」「Doing」「Done」の3カラム（カンバン形式）で構成される。
2. **タスク追加**: サイドバーまたは画面上部のフォームから、タスク詳細（タイトル、優先度、担当者）を入力し「追加」ボタンを押下。タスクが「To Do」カラムに表示される。
3. **ステータス変更**: 各タスクカード内のセレクトボックス操作によりステータスを変更。変更後、即座に該当カラムへ移動する。
4. **タスク削除**: 各タスクカード内の「削除」ボタンを押下。即座に画面およびDBからタスクが削除される。

### <Verify>
1. **UI構成の実証**: Streamlitの `st.columns(3)` を使用することで、3カラムのカンバンUIを構築することが可能。
2. **DBアクセスの整合性**: Streamlitはインタラクションのたびにスクリプト全体を再実行する仕様。このため、再実行の最初にDBから最新のデータを取得すれば状態の整合性は保たれる。単一ユーザーのローカルツールであるため、`sqlite3` ライブラリの標準的なCRUD操作で並行書き込み問題は発生しない。
3. **状態変更と再描画**: ボタン押下やセレクトボックスの変更でスクリプトが再実行される。その際、ウィジェットのコールバック(`on_change` や `on_click`)を活用して先にDBを更新することで、次回描画時に最新データが反映される。必要に応じて `st.rerun()` を使用する。
